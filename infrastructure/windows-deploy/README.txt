================================================================================
  Loyalty BFF — установка на Windows Server 2016+
================================================================================

Этот пакет содержит скрипты для установки и обслуживания BFF (Backend For
Frontend) приложения лояльности на сервере под управлением Windows Server.
Используется Node.js напрямую (без Docker), сервис регистрируется через NSSM.

--------------------------------------------------------------------------------
  СОДЕРЖИМОЕ ПАКЕТА
--------------------------------------------------------------------------------

  install.bat       Однокликовый установщик (cmd)
  install.ps1       То же самое, но на PowerShell
  start.bat         Запуск сервиса (или ручной запуск через "start.bat manual")
  stop.bat          Остановка сервиса
  update.bat        Обновление до последнего коммита: git pull + сборка + рестарт
  .env.template     Шаблон конфигурации — скопировать в apps\bff\.env
  README.txt        Этот файл

--------------------------------------------------------------------------------
  ПРЕДВАРИТЕЛЬНЫЕ ТРЕБОВАНИЯ
--------------------------------------------------------------------------------

Перед запуском install.bat убедитесь, что на сервере установлено:

  1. Node.js 20.x LTS
     https://nodejs.org/dist/latest-v20.x/
     Скачайте .msi установщик для Windows x64 и запустите.
     Проверка: открыть cmd и выполнить "node -v" — должно вывести v20.x.x

  2. Git for Windows
     https://git-scm.com/download/win
     При установке выбрать "Git from the command line and also from 3rd-party
     software". Проверка: "git --version".

  3. PostgreSQL 14+
     https://www.postgresql.org/download/windows/
     Создать базу loyalty_db и пользователя loyalty с паролем.
     Запомните строку подключения для DATABASE_URL в .env.

  4. Memurai (или Redis под WSL2)
     https://www.memurai.com/get-memurai
     Memurai — это Redis-совместимая БД, официально поддерживаемая на Windows.
     Бесплатная Developer Edition подходит. По умолчанию слушает 127.0.0.1:6379.

  5. NSSM (Non-Sucking Service Manager)
     https://nssm.cc/download
     Скачать ZIP, распаковать, скопировать nssm.exe (из подпапки win64\)
     В ЭТУ ЖЕ ПАПКУ (рядом с install.bat).

--------------------------------------------------------------------------------
  ПОШАГОВАЯ УСТАНОВКА
--------------------------------------------------------------------------------

ШАГ 1. Подключитесь к серверу по RDP под учетной записью с правами
       администратора.

ШАГ 2. Установите все компоненты из раздела "Предварительные требования".

ШАГ 3. Склонируйте репозиторий проекта в произвольную папку, например
       C:\loyalty-app:

           cd C:\
           git clone https://github.com/dnkfood/loyalty-app.git
           cd loyalty-app

       Альтернатива: распаковать ZIP-архив проекта в C:\loyalty-app.

ШАГ 4. Скопируйте nssm.exe в папку infrastructure\windows-deploy\.

ШАГ 5. Скопируйте шаблон конфигурации и заполните реальными значениями:

           copy infrastructure\windows-deploy\.env.template apps\bff\.env
           notepad apps\bff\.env

       Обязательно укажите:
         - DATABASE_URL          (строка подключения к PostgreSQL)
         - REDIS_URL             (обычно redis://localhost:6379)
         - JWT_ACCESS_SECRET     (случайные 64 hex-символа)
         - JWT_REFRESH_SECRET    (случайные 64 hex-символа, отличающиеся)
         - LOYALTY_WEBHOOK_SECRET (случайные hex-символы)
         - ALFA_SMS_API_KEY      (получить у провайдера AlfaSMS)
         - CORS_ORIGINS          (домены админки через запятую)

       Для генерации случайных секретов в PowerShell:
           [System.BitConverter]::ToString(
             (1..32 | ForEach-Object { Get-Random -Max 256 }) -as [byte[]]
           ).Replace('-','').ToLower()

ШАГ 6. Откройте cmd ОТ ИМЕНИ АДМИНИСТРАТОРА (правый клик -> "Запуск от
       имени администратора") и перейдите в папку со скриптами:

           cd C:\loyalty-app\infrastructure\windows-deploy

ШАГ 7. Запустите установщик:

           install.bat

       Скрипт проверит окружение, установит pnpm (если нужно), соберёт
       все зависимости, применит миграции базы данных и зарегистрирует
       Windows-сервис LoyaltyBff с автозапуском.

       Альтернатива на PowerShell (от имени администратора):
           Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass
           .\install.ps1

ШАГ 8. Проверьте, что сервис запущен:

           sc query LoyaltyBff

       Состояние должно быть RUNNING. Логи лежат в logs\bff-stdout.log и
       logs\bff-stderr.log в этой же папке.

ШАГ 9. Откройте брандмауэр Windows для внешних подключений к BFF
       (порт по умолчанию 3000):

           netsh advfirewall firewall add rule name="LoyaltyBff" ^
                 dir=in action=allow protocol=TCP localport=3000

--------------------------------------------------------------------------------
  УПРАВЛЕНИЕ СЕРВИСОМ
--------------------------------------------------------------------------------

  start.bat            Запустить сервис LoyaltyBff
  start.bat manual     Запустить BFF в текущей консоли (для отладки)
  stop.bat             Остановить сервис
  update.bat           Обновить код, пересобрать, применить миграции, рестарт

Просмотр логов:
  type logs\bff-stdout.log
  type logs\bff-stderr.log

Управление через стандартные средства Windows:
  services.msc -> найти "Loyalty App BFF (NestJS)"

Удаление сервиса вручную (если что-то пошло не так):
  nssm.exe stop LoyaltyBff
  nssm.exe remove LoyaltyBff confirm

--------------------------------------------------------------------------------
  ОБНОВЛЕНИЕ
--------------------------------------------------------------------------------

После того как разработчик запушил новый коммит в master:

  cd C:\loyalty-app\infrastructure\windows-deploy
  update.bat

Скрипт остановит сервис, выполнит git pull, пересоберёт пакеты, применит
новые миграции базы данных и снова запустит сервис. Если на каком-либо
шаге будет ошибка, сервис останется остановленным — проверьте логи и
запустите install.bat повторно.

--------------------------------------------------------------------------------
  УСТРАНЕНИЕ НЕПОЛАДОК
--------------------------------------------------------------------------------

[ERROR] This script must be run as Administrator
  Запустите cmd через "Запуск от имени администратора".

[ERROR] Node.js is not installed or not in PATH
  Установите Node.js 20.x LTS и перезапустите cmd (PATH обновляется только
  в новых сессиях).

[ERROR] nssm.exe not found
  Скачайте NSSM с https://nssm.cc/download, распакуйте и положите файл
  win64\nssm.exe в эту же папку (infrastructure\windows-deploy\).

[ERROR] prisma migrate deploy failed
  Проверьте DATABASE_URL в apps\bff\.env. Попробуйте подключиться к базе
  командой psql или DBeaver. Убедитесь, что пользователь имеет права
  CREATE на схему public.

[ERROR] Failed to start service
  Откройте logs\bff-stderr.log — там будет ошибка из Node.js. Частые
  причины: не найден REDIS_URL, неверный JWT_*_SECRET, отсутствует папка
  dist (значит сборка не прошла — запустите install.bat ещё раз).

Сервис стартует, но возвращает 500 на запросах
  Это уже не проблема установки. Смотрите logs\bff-stderr.log в режиме
  реального времени:
      powershell -Command "Get-Content -Path logs\bff-stderr.log -Wait"

--------------------------------------------------------------------------------
  ВАЖНЫЕ ЗАМЕЧАНИЯ
--------------------------------------------------------------------------------

* Скрипты НЕ устанавливают системные компоненты автоматически (Node.js,
  PostgreSQL, Memurai, Git, NSSM). Это сделано намеренно, чтобы избежать
  непредсказуемых изменений на сервере. Установите их вручную один раз.

* Файл apps\bff\.env содержит секреты. НЕ коммитьте его в git и
  ограничьте доступ к нему на уровне NTFS-разрешений (только для учетной
  записи, под которой работает сервис, и администраторов).

* Сервис по умолчанию запускается под учетной записью LocalSystem. Если
  нужна отдельная учетная запись, используйте:
      nssm.exe set LoyaltyBff ObjectName ".\loyalty-svc" "пароль"

* Для смены порта измените PORT в apps\bff\.env, выполните stop.bat и
  start.bat, и обновите правило брандмауэра под новый порт.

* Версия Node.js должна быть строго v20.x.x. Версии v18 или v22 могут
  работать, но не тестировались — установщик выведет предупреждение.
